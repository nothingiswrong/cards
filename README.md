## Симуляция


|                                                                                                                             |                                                                                                                                             |
| --------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| Базовый класс карты: название, номер; метод применения к сумме (`Use`), при невозможности - `false`; сумма покупки меняется | `ru.cards.models.Card` + `use(Purchase purchase)`; покупка - `Purchase.decreaseSum`                                                         |
| Дисконтная и дебетовая карты                                                                                                | `DiscountCard`, `DebetCard`                                                                                                                 |
| Накопительная дисконтная и подарочная (без пополнения баланса подарка)                                                      | `AccumulativeDiscountCard`, `GiftCard` + `GiftCardStatus`                                                                                   |
| Набор карт, случайный выбор, попытки до оплаты                                                                              | `SimulationRunner.generateCards()`, `useCard` с `ThreadLocalRandom`, цикл до `PAID` или лимита шагов                                        |
| Покупки до сценария "невозможно"                                                                                            | Проверка `checkIfPurchasePossible` - завершение с `IMPOSSIBLE_TO_MAKE_PURCHASE`; либо исчерпание шагов - `EXCEEDED_MAX_NUMBER_OF_PURCHASES` |
| Визуализация (GUI)                                                                                                          | Отдельное приложение `web/`, данные с API                                                                                                   |


### Модель карт

- `**Card.use(Purchase)`** - в базовом классе возвращает `false`; наследники уменьшают `Purchase.remainingAmount` через `**Purchase.decreaseSum`** (с проверками и переходом в `PAID` при нулевом остатке).
- `**DiscountCard**` - списывает не больше фиксированной скидки и не больше остатка; успех при наличии чего списать.
- `**DebetCard**` - если баланс **>=** остатка покупки, закрывает остаток одним применением и уменьшает баланс; иначе `**false`**.
- `**GiftCard`** - одно успешное применение в статусе `VALID` (до номинала или остатка), затем `INVALID`; пополнения нет.
- `**AccumulativeDiscountCard**` - списание ограничено текущей скидкой и остатком; после использования скидка может расти шагом до потолка.

### Покупка

`**Purchase**` хранит идентификатор, исходную сумму, остаток, статус `UNPAID` / `PAID`. Повторное списание после оплаты запрещено (исключение).

### Симуляция (`SimulationRunner`)

1. Генерируется кошелёк: **6-20** случайных карт четырёх типов (`generateCards`).
2. В цикле до `**MAX_PURCHASES_SESSION`** (20) создаётся покупка со случайной суммой **100 ... 14 999** (`nextInt(100, 15_000)`).
3. `**checkIfPurchasePossible`** - быстрая оценка: при наличии дисконтной/накопительной считается, что сценарий потенциально разрешим; иначе сумма **действующих** подарочных + **максимальный** баланс дебета сравнивается с суммой покупки (модель "подарки + один дебет на остаток"). При `false` возвращается сессия с одним `PurchaseResult` и видом `**IMPOSSIBLE_TO_MAKE_PURCHASE`**.
4. До `**MAX_PAY_STEPS_PER_PURCHASE`** (100) случайно выбирается карта, вызывается `**use**`. Фиксируется `**SimulationStep**` (карта, покупка, дельта по остатку, `**SUCCESS**` / `**FAILURE**`). При `**PAID**` цикл прерывается, `**PURCHASE_MADE**`. Иначе по окончании попыток - `**EXCEEDED_MAX_NUMBER_OF_PURCHASES**` (по смыслу лимит **попыток применения карт**, не числа покупок в сессии).
